# ADR 0001 — 采用架构决策记录（ADR）

> 状态：已接受 · 日期：2026-06-02

## 背景

autofresh 已经积累了一些隐含的关键决策（例如：报告只读本机、不输出配额百分比、保活用
`--ephemeral` 不写 rollout）。这些「为什么这样做」散落在代码注释与 README 里，
新功能（AI 用量分析报告）会引入更多需要权衡的选型。缺少统一记录会导致重复讨论、
决策被无意推翻。

## 决策

采用 **ADR（Architecture Decision Record）** 记录影响面大或不可逆的技术决策：

- 每个决策一篇 Markdown，放在 `docs/adr/`，文件名 `NNNN-短标题.md`，编号递增。
- 每篇包含：**背景 / 决策 / 后果（取舍）**，可选「备选方案」。
- 状态取值：`提议中` → `已接受` / `已废弃` / `被 NNNN 取代`。
- 已接受的 ADR **只增不改**；改变主意时新增一篇并标注取代关系。

## 后果

- 好处：决策可追溯，评审有据，新人能快速理解「为什么不这么做」。
- 成本：做关键决策时多写一篇短文档。
- 范围：仅记录「关键」决策，日常实现细节仍走代码与注释，避免文档膨胀。
