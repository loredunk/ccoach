# ADR 0061 — 项目体检重定义为「阶段门控的盲区教练」：教练不是审计员

> 状态：已接受 · 日期：2026-06-12 · 分支：`0.1.7`
> · **修订 [`adr/0060`](0060-deepinsight-project-health-check-beta.md)**：0060 的四维隐私边界、Beta 状态、
>   本地限定保留；其 D1/D3 的「0-4 评分 + 重构阈值」审计式语义被本篇取代（ADR 只增不改，特此另立）
> · 与 [`adr/0006`](0006-feature-first-recommendations.md)（feature-first）同一纪律：修复动作导回 harness

## 背景

0060 落地后用户重新定义了这个特性的产品立场，三条原则（均为用户原话的提炼）：

1. **教练不是审计员。** 0-4 评分是审计结论（「你的安全 2 分」），ccoach 要背「说了没问题结果出事」的
   责任。教练只指出盲区、布置作业，**审计本身交给 agent 下个会话执行**（「你从没碰过鉴权这个维度，
   下个会话让 Claude 跑一遍 security review，重点看这三个文件」）——修复动作导回 harness 本身，与
   deepinsight「每个建议落到官方特性」一致。
2. **阶段门控，不然就是噪音。** 对周末原型喊「你缺可观测性和灾备」是制造焦虑。必须先判项目阶段
   （提交频率、是否有 CI、是否提到过部署/用户），原型期只报安全红线级盲区，有真实用户信号后才解锁
   完整维度。「什么阶段该懂什么」的成熟度模型本身就是教练产品的内容资产。
3. **缺席断言，不做在场断言。** 「你没写测试」在测试放非常规目录时就是误报，一次误报毁掉整个特性的
   信任。措辞必须是「这个维度从未在你的会话中出现」（可 grep 验证的行为事实），而不是「你的项目缺
   这个」（不可证实的审计判断）。

## 决策

### D1 schema v2：stage + attention 取代 score

- 新增 `project_health.stage`：`level: prototype | shipping | serving` + `label` + `basis`（判定所依据
  的可验证信号原文）+ `note`。阶段外维度标 `locked: true`，渲染为安静的「此阶段未解锁」行（无作业）——
  锁定行可见，让成熟度模型本身成为内容。
- 维度字段：`attention: never | touched | practiced`（该维度在本窗口会话/提交中的出现频度——行为覆盖，
  不是项目质量）+ `statement`（可验证的行为/在场事实）+ `basis`（怎么查的）+ `homework`（交给 agent 的
  作业）+ `feature`（官方特性药丸）。**省略 attention = 未核查**（虚线空轨道），沿袭 0060「省略=诚实」
  的结构化语义；枚举校验（`Object.hasOwn`），非法值降级为未核查、绝不伪装成档位。
- 删除 0-4 `score`/`level 档位词`/`threshold` 字段。架构维度的真实数字（「~1,400 行、被 5 模块引用」）
  是**在场事实**，进 `statement`/`basis`；「该不该拆」的判断变成 homework（让 agent 做重构评估）。

### D2 措辞红线（honesty rules 的延伸）

statement 只允许可验证的行为/在场事实：「X 从未在本窗口会话/提交中出现」「repo 无 CI 配置文件」
「发现一处硬编码凭据（在场事实，值绝不写出）」。禁止任何项目质量断言（「鉴权薄弱」「可能内存泄漏」）。
缺席断言比在场断言难误报——grep 即可复核；这是该特性能维持信任的根本。

### D3 渲染 v2

进度轨道按 attention 填充（never 0% 红 / touched 50% 琥珀 / practiced 100% 绿；档位词同色）；stage
横幅（判定 + 依据 + 门控说明）置于区块顶部；locked 行灰显无作业；homework 复用 fix 高亮块 + feature
药丸。Beta 徽章与本地限定不变；目录行随区块渲染必然成对（单一谓词，先渲染区块再判 TOC）。

### D4 边界沿袭

0060 的隐私边界全部保留：本地 HTML 限定、绝不进可分享产物（SKILL.md 分享路径现明确「分享前先从 JSON
删除 project_health」）、脱敏与 basename 口径、密钥值绝不落字段、CLI 零改动双平台对称。

## 后果

- ccoach 不再输出任何「项目质量评级」，不背审计责任；输出物变为「盲区 + 作业」，天然导流回 harness 的
  官方特性（security review、hook、子代理审计）。
- 阶段门控需要读取会话提示词/提交信息做关键词命中统计——均在既有隐私授权内（本人 prompt 默认读，
  ADR 0015；提交信息本地 git）。
- 阶段阈值与 attention 分桶仍是 Beta，校准后再去 Beta（TODO 跟踪）。
