# ADR 0008 — 游戏化等级与可分享成绩卡（病毒传播）

> 状态：已接受（已实现） · 日期：2026-06-02 · 相关：[ADR 0004](0004-skills-based-analysis.md)（模型写吐槽文案）、
> [ADR 0005](0005-tiered-analysis-and-signals.md)（prompt 分析、本地隐私）、
> [ADR 0006](0006-feature-first-recommendations.md)、[`PRD.md`](../PRD.md) §3.11、[`TODO.md`](../TODO.md) T7

## 背景

ccoach 用 HTML 呈现用量分析，天然适合做「**情绪价值 + 社交传播**」。对标 ccusage——它靠
「Claude Code cost scorecard」病毒式传播起来（官方都称那条视频为 "the Claude Code cost
scorecard that went viral"）。ccoach 比它**多了习惯与 prompt 两个维度**，等级体系能玩得更深。

人愿意截图分享的心理是两种：**炫耀**（我是大师）和**自嘲**（我是菜鸟，太真实了）。两端都要能
让人主动发出去，因此菜鸟段位也要有梗、好笑、不羞辱人。单一总分会让大多数人落在中间、没分享欲；
**多维度独立评级**才能让每个人在某条轴上拿到一个「能炫或能笑」的标签。

## 决策

### D1 — 多轴独立评级（不做单一总分）

3~4 条独立的轴，每条有自己的段位名 + 一句吐槽文案：

- **Prompt 功力**（核心轴，最具传播力——这是 ccoach 独有的 prompt 分析维度）：依据 prompt 长度、
  结构化程度、约束清晰度、中途改需求频率。段位：大师级 / 老练 / 学徒 / 复读机 / 玄学召唤师。
- **烧钱姿势**（对应成本，但加「值不值」判断）：结合花费与产出效率。段位：性价比刺客 /
  理性消费 / 富哥随意 / Opus 锤钉子（拿最贵模型干最简单的活）。
- **工程素养**（git / session 习惯轴）：session 收敛速度、是否善用 plan、改动是否聚焦。
  段位：架构师 / 工程师 / 莽夫 / 考古学家（一个 session 跨三个无关项目）。
- **勤奋度**（用量频率轴，纯娱乐）：日活、连续天数、深夜占比。段位：劳模 / 996 战士 /
  养生程序员 / 周末才想起来。

### D2 — 可单独截图的「封面成绩卡」

HTML 报告很长，但顶部放一张**设计好看、可单独截图**的成绩卡：几条轴的段位 + 一句最扎心的
总结文案，**竖版、适配手机截图、信息密度适中**。这张图是被转发的主体，下面的详细分析是给本人看的。

### D3 — 可分享的「人格总结」由模型生成

把几条轴合成一句略带羞耻的称号（如「玄学召唤师 × Opus 锤钉子 × 考古学家」），让 Claude / Codex
读用量数据后写一段**又准又毒**的吐槽。这正好落在 [ADR 0004](0004-skills-based-analysis.md)
的 skills 化路径上（模型在用户当前 agent 侧生成，不在二进制内）。

### D4 — 留对比钩子（相对排名）

「超过了 73% 的用户」这类相对排名最能刺激分享。早期没有真实大盘数据时，先用**基于合理假设的
本地估算**，后期有数据再校准（并标注为估算）。

### D5 — 分寸：「损」但不伤人

吐槽**行为习惯（可改变的）**，绝不攻击能力或人格。可以「你的 prompt 像在跟 AI 玩猜谜」，
不可以「你不适合编程」。前者让人笑着改进，后者让人删 app。

### D6 — 隐私即卖点

prompt 内容分析全程**本地处理、绝不上传**（沿用 [ADR 0005](0005-tiered-analysis-and-signals.md)
的本地 + 脱敏 + 不读 assistant 回复）。在 README 与 UI 明确写出「所有分析在本地完成，prompt
内容不离开你的机器」——这本身是信任卖点，反而促进传播。

## 后果

- 好处：把枯燥的用量数据变成可炫 / 可自嘲的社交货币，借鉴 ccusage 的病毒路径并以习惯 / prompt
  维度做出差异化；与既有 skills 化（D3）、本地隐私（D6）天然契合。
- 代价：段位阈值与文案需打磨（分寸见 D5）；相对排名早期是估算，需明确标注；成绩卡版式要专门设计。
- 影响：在 `skills/ai-usage-html-report/` 的渲染层新增「封面成绩卡」，并新增一份段位 / 文案 reference。
